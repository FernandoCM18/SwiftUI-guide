## Tipos fundamentales

Swift proporciona un conjunto rico de tipos de datos integrados. Todos los tipos en Swift son tipos de valor o tipos de referencia.

## Enteros: Int

Los enteros son números sin parte decimal. Swift ofrece `Int` (con signo) y `UInt` (sin signo).

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let edad: Int = 25
    let poblacion = 7_900_000_000  // Guiones bajos para legibilidad
    let negativo = -42

    // Límites del tipo
    print("Int mínimo: \(Int.min)")
    print("Int máximo: \(Int.max)")

    // Otros tamaños: Int8, Int16, Int32, Int64
    let pequeño: Int8 = 127  // máximo de Int8
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const edad: number = 25;
    const poblacion = 7_900_000_000;  // Guiones bajos para legibilidad
    const negativo = -42;

    // Límites del tipo (no hay Int separado, todo es number)
    console.log(`Entero seguro mín: ${Number.MIN_SAFE_INTEGER}`);
    console.log(`Entero seguro máx: ${Number.MAX_SAFE_INTEGER}`);

    // Para enteros grandes, usar BigInt
    const grande: bigint = 9007199254740993n;
    ```
  </TabItem>
</Tabs>

## Decimales: Double y Float

`Double` tiene precisión de 64 bits (15 dígitos). `Float` tiene precisión de 32 bits (6 dígitos). Swift infiere `Double` por defecto.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let pi: Double = 3.14159265358979
    let gravedad: Float = 9.81

    // Swift infiere Double automáticamente
    let temperatura = 36.6  // es Double, no Float

    // Operaciones con decimales
    let radio = 5.0
    let area = pi * radio * radio
    print("Área del círculo: \(area)")
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const pi: number = 3.14159265358979;
    const gravedad: number = 9.81;

    // TS solo tiene number (64-bit float, equivalente a Double)
    const temperatura = 36.6;

    // Operaciones con decimales
    const radio = 5.0;
    const area = pi * radio * radio;
    console.log(`Área del círculo: ${area}`);
    ```
  </TabItem>
</Tabs>

## Booleanos: Bool

Los booleanos solo tienen dos valores: `true` o `false`.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let esMayorDeEdad = true
    let tienePermiso = false

    // Operadores lógicos
    let puedeEntrar = esMayorDeEdad && tienePermiso  // AND: false
    let tieneAcceso = esMayorDeEdad || tienePermiso   // OR: true
    let noEsMenor = !esMayorDeEdad                    // NOT: false

    print("¿Puede entrar? \(puedeEntrar)")
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const esMayorDeEdad = true;
    const tienePermiso = false;

    // Operadores lógicos (idénticos a Swift)
    const puedeEntrar = esMayorDeEdad && tienePermiso;  // AND: false
    const tieneAcceso = esMayorDeEdad || tienePermiso;   // OR: true
    const noEsMenor = !esMayorDeEdad;                    // NOT: false

    console.log(`¿Puede entrar? ${puedeEntrar}`);
    ```
  </TabItem>
</Tabs>

## Cadenas: String

Las cadenas de texto son colecciones de caracteres Unicode.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let saludo = "¡Hola, mundo!"
    let multilínea = """
        Esta es una cadena
        que ocupa varias
        líneas de texto.
        """

    // Operaciones comunes
    let nombre = "Swift"
    print(nombre.count)           // 5 - longitud
    print(nombre.uppercased())    // SWIFT
    print(nombre.lowercased())    // swift
    print(nombre.isEmpty)         // false
    print(nombre.contains("wi"))  // true

    // Concatenación
    let lenguaje = "Swift"
    let version = "6"
    let completo = lenguaje + " " + version  // "Swift 6"
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const saludo = "¡Hola, mundo!";
    const multilinea = `Esta es una cadena
    que ocupa varias
    líneas de texto.`;

    // Operaciones comunes
    const nombre = "Swift";
    console.log(nombre.length);            // 5 - longitud
    console.log(nombre.toUpperCase());     // SWIFT
    console.log(nombre.toLowerCase());     // swift
    console.log(nombre.length === 0);      // false
    console.log(nombre.includes("wi"));    // true

    // Concatenación y template literals
    const lenguaje = "Swift";
    const version = "6";
    const completo = `${lenguaje} ${version}`;  // "Swift 6"
    ```
  </TabItem>
</Tabs>

## Caracteres: Character

Un `Character` representa un solo carácter Unicode.

```swift

let acento: Character = "ñ"

// Iterar caracteres de un String
let palabra = "Hola"
for caracter in palabra {
    print(caracter)
}
```

## Arrays

Los arrays son colecciones ordenadas de valores del mismo tipo.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    // Crear arrays
    var frutas: [String] = ["Manzana", "Banana", "Cereza"]
    var numeros = [1, 2, 3, 4, 5]  // inferencia de tipo
    var vacio: [Int] = []

    // Acceder y modificar
    print(frutas[0])          // "Manzana"
    frutas.append("Durazno")  // Agregar al final
    frutas.insert("Fresa", at: 1)  // Insertar en posición
    frutas.remove(at: 0)      // Eliminar elemento

    // Propiedades útiles
    print(frutas.count)        // Número de elementos
    print(frutas.isEmpty)      // ¿Está vacío?
    print(frutas.first ?? "")  // Primer elemento (opcional)
    print(frutas.last ?? "")   // Último elemento (opcional)

    // Iterar
    for fruta in frutas {
        print(fruta)
    }
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    // Crear arrays
    let frutas: string[] = ["Manzana", "Banana", "Cereza"];
    let numeros = [1, 2, 3, 4, 5];  // inferencia de tipo
    let vacio: number[] = [];

    // Acceder y modificar
    console.log(frutas[0]);             // "Manzana"
    frutas.push("Durazno");             // Agregar al final
    frutas.splice(1, 0, "Fresa");       // Insertar en posición
    frutas.splice(0, 1);                // Eliminar elemento

    // Propiedades útiles
    console.log(frutas.length);          // Número de elementos
    console.log(frutas.length === 0);    // ¿Está vacío?
    console.log(frutas.at(0) ?? "");     // Primer elemento
    console.log(frutas.at(-1) ?? "");    // Último elemento

    // Iterar
    for (const fruta of frutas) {
      console.log(fruta);
    }
    ```
  </TabItem>
</Tabs>

## Diccionarios: Dictionary

Los diccionarios almacenan pares clave-valor sin orden definido.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    // Crear diccionarios
    var capitales: [String: String] = [
        "España": "Madrid",
        "Francia": "París",
        "Italia": "Roma"
    ]

    // Acceder a valores (devuelve opcional)
    if let capital = capitales["España"] {
        print("La capital de España es \(capital)")
    }

    // Modificar
    capitales["Alemania"] = "Berlín"       // Agregar
    capitales["Francia"] = "Lyon"          // Actualizar (ejemplo)
    capitales.removeValue(forKey: "Italia") // Eliminar

    // Iterar
    for (pais, capital) in capitales {
        print("\(pais): \(capital)")
    }

    // Propiedades
    print(capitales.keys)    // Todas las claves
    print(capitales.values)  // Todos los valores
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    // Crear objetos (equivalente más común a Dictionary)
    const capitales: Record<string, string> = {
      España: "Madrid",
      Francia: "París",
      Italia: "Roma",
    };

    // Acceder a valores
    const capital = capitales["España"];
    if (capital) {
      console.log(`La capital de España es ${capital}`);
    }

    // Modificar
    capitales["Alemania"] = "Berlín";       // Agregar
    capitales["Francia"] = "Lyon";          // Actualizar (ejemplo)
    delete capitales["Italia"];              // Eliminar

    // Iterar
    for (const [pais, capital] of Object.entries(capitales)) {
      console.log(`${pais}: ${capital}`);
    }

    // Propiedades
    console.log(Object.keys(capitales));    // Todas las claves
    console.log(Object.values(capitales));  // Todos los valores
    ```
  </TabItem>
</Tabs>

## Conjuntos: Set

Los conjuntos almacenan valores únicos sin orden definido.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    // Crear conjuntos
    var colores: Set<String> = ["Rojo", "Verde", "Azul"]
    var numeros: Set = [1, 2, 3, 4, 5]

    // No permite duplicados
    colores.insert("Rojo")  // No se agrega, ya existe
    colores.insert("Amarillo")  // Se agrega

    // Operaciones de conjuntos
    let pares: Set = [2, 4, 6, 8]
    let primos: Set = [2, 3, 5, 7]

    let union = pares.union(primos)                // Todos los elementos
    let interseccion = pares.intersection(primos)  // Elementos comunes: {2}
    let diferencia = pares.subtracting(primos)     // En pares pero no en primos

    print("Unión: \(union)")
    print("Intersección: \(interseccion)")
    print("Diferencia: \(diferencia)")
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    // Crear conjuntos
    const colores = new Set<string>(["Rojo", "Verde", "Azul"]);
    const numeros = new Set([1, 2, 3, 4, 5]);

    // No permite duplicados
    colores.add("Rojo");       // No se agrega, ya existe
    colores.add("Amarillo");   // Se agrega

    // Operaciones de conjuntos
    const pares = new Set([2, 4, 6, 8]);
    const primos = new Set([2, 3, 5, 7]);

    const union = new Set([...pares, ...primos]);
    const interseccion = new Set([...pares].filter((x) => primos.has(x)));
    const diferencia = new Set([...pares].filter((x) => !primos.has(x)));

    console.log("Unión:", union);
    console.log("Intersección:", interseccion);
    console.log("Diferencia:", diferencia);
    ```
  </TabItem>
</Tabs>

## Tuplas

Las tuplas agrupan varios valores en un solo valor compuesto.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    // Tupla simple
    let coordenada = (x: 10.0, y: 20.0)
    print(coordenada.x)  // 10.0
    print(coordenada.y)  // 20.0

    // Tupla sin nombres
    let httpError = (404, "No encontrado")
    print(httpError.0)  // 404
    print(httpError.1)  // "No encontrado"

    // Descomponer tuplas
    let (codigo, mensaje) = httpError
    print("Error \(codigo): \(mensaje)")

    // Ignorar valores con _
    let (soloCodigo, _) = httpError
    print("Código: \(soloCodigo)")

    // Tupla como retorno de función
    func dividir(_ a: Int, entre b: Int) -> (cociente: Int, resto: Int) {
        return (a / b, a % b)
    }
    let resultado = dividir(17, entre: 5)
    print("Cociente: \(resultado.cociente), Resto: \(resultado.resto)")
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    // Tupla con tipo explícito
    const coordenada: [number, number] = [10.0, 20.0];
    console.log(coordenada[0]);  // 10.0
    console.log(coordenada[1]);  // 20.0

    // Tupla con tipos mixtos
    const httpError: [number, string] = [404, "No encontrado"];
    console.log(httpError[0]);  // 404
    console.log(httpError[1]);  // "No encontrado"

    // Desestructurar tuplas
    const [codigo, mensaje] = httpError;
    console.log(`Error ${codigo}: ${mensaje}`);

    // Ignorar valores
    const [soloCodigo] = httpError;
    console.log(`Código: ${soloCodigo}`);

    // Tupla como retorno de función
    function dividir(a: number, b: number): { cociente: number; resto: number } {
      return { cociente: Math.floor(a / b), resto: a % b };
    }
    const resultado = dividir(17, 5);
    console.log(`Cociente: ${resultado.cociente}, Resto: ${resultado.resto}`);
    ```
  </TabItem>
</Tabs>

<PlaygroundLink />

:::tip
Las tuplas son ideales para retornar múltiples valores de una función. Para estructuras de datos más complejas, usa `struct`.
:::

## Conversión de tipos

Swift no convierte tipos automáticamente. Debes hacerlo de forma explícita.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let entero = 42
    let decimal = Double(entero)    // Int → Double
    let texto = String(entero)      // Int → String
    let numero = Int("123") ?? 0    // String → Int (opcional)

    let a: Int = 10
    let b: Double = 3.5
    let suma = Double(a) + b       // Convertir antes de operar
    print("Suma: \(suma)")
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const entero = 42;
    const decimal = Number(entero);       // ya es number, no-op
    const texto = String(entero);         // number → string
    const numero = parseInt("123") || 0;  // string → number

    const a = 10;
    const b = 3.5;
    const suma = a + b;  // No necesita conversión, todo es number
    console.log(`Suma: ${suma}`);
    ```
  </TabItem>
</Tabs>

## Resumen

| Tipo | Ejemplo | Descripción |
|------|---------|-------------|
| `Int` | `42` | Número entero |
| `Double` | `3.14` | Decimal de 64 bits |
| `Float` | `9.81` | Decimal de 32 bits |
| `Bool` | `true` | Verdadero o falso |
| `String` | `"Hola"` | Cadena de texto |
| `Character` | `"A"` | Un carácter |
| `[Tipo]` | `[1, 2, 3]` | Array |
| `[K: V]` | `["a": 1]` | Diccionario |
| `Set<Tipo>` | `Set([1, 2])` | Conjunto |
| `(T1, T2)` | `(1, "a")` | Tupla |
