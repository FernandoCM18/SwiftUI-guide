## Estructuras de control

Las estructuras de control permiten dirigir el flujo de ejecución de tu programa. Swift ofrece todas las estructuras clásicas con mejoras modernas como el pattern matching.

## Condicional if/else

La estructura más básica para tomar decisiones.

<Tabs syncKey="lang">
  <TabItem label="Swift">

        print("Hace frío")

### Condiciones múltiples

Puedes combinar condiciones con operadores lógicos.

```swift
let edad = 25
let tieneEntrada = true

// AND (&&): ambas deben ser verdaderas
if edad >= 18 && tieneEntrada {
    print("Puedes entrar al concierto")
}

// OR (||): al menos una debe ser verdadera
let esEstudiante = false
let esMayorDe65 = false
if esEstudiante || esMayorDe65 {
    print("Tienes descuento")
} else {
    print("Precio normal")
}
```

## Sentencia switch

El `switch` de Swift es muy potente. Debe ser exhaustivo y no necesita `break` (no hay fall-through implícito).

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    let dia = "lunes"

    switch dia {
    case "lunes":
        print("Inicio de semana")
    case "martes", "miércoles", "jueves":
        print("Entre semana")
    case "viernes":
        print("¡Por fin viernes!")
    case "sábado", "domingo":
        print("Fin de semana")
    default:
        print("Día no reconocido")
    }
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    const dia = "lunes";

    switch (dia) {
      case "lunes":
        console.log("Inicio de semana");
        break;
      case "martes":
      case "miércoles":
      case "jueves":
        console.log("Entre semana");
        break;
      case "viernes":
        console.log("¡Por fin viernes!");
        break;
      case "sábado":
      case "domingo":
        console.log("Fin de semana");
        break;
      default:
        console.log("Día no reconocido");
    }
    ```
  </TabItem>

### Pattern matching con rangos

```swift
let nota = 85

switch nota {
case 0..<50:
    print("Suspenso")
case 50..<60:
    print("Aprobado")
case 60..<70:
    print("Bien")
case 70..<90:
    print("Notable")
case 90...100:
    print("Sobresaliente")
default:
    print("Nota fuera de rango")
}
```

<PlaygroundLink />

### Pattern matching con tuplas

```swift
let punto = (2, 0)

switch punto {
case (0, 0):
    print("En el origen")
case (_, 0):
    print("En el eje X")
case (0, _):
    print("En el eje Y")
case (-2...2, -2...2):
    print("Dentro del cuadrado central")
default:
    print("Fuera del cuadrado central")
}
```

### Value binding en switch

```swift
let otroPunto = (3, -5)

switch otroPunto {
case (let x, 0):
    print("En el eje X con valor \(x)")
case (0, let y):
    print("En el eje Y con valor \(y)")
case let (x, y) where x == y:
    print("En la diagonal donde x == y == \(x)")
case let (x, y):
    print("En algún lugar: (\(x), \(y))")
}
```

## Bucle for-in

El bucle `for-in` itera sobre secuencias como rangos, arrays y diccionarios.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    // Iterar sobre un rango
    for i in 1...5 {
        print("Número: \(i)")
    }

    // Rango que excluye el límite superior
    for i in 0..<3 {
        print("Índice: \(i)")
    }

    // Iterar sobre un array
    let frutas = ["Manzana", "Banana", "Cereza"]
    for fruta in frutas {
        print("Fruta: \(fruta)")
    }

    // Iterar con índice usando enumerated()
    for (indice, fruta) in frutas.enumerated() {
        print("\(indice): \(fruta)")
    }

    // Iterar sobre un diccionario
    let edades = ["Ana": 25, "Luis": 30, "María": 28]
    for (nombre, edad) in edades {
        print("\(nombre) tiene \(edad) años")
    }
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    // Iterar sobre un rango
    for (let i = 1; i <= 5; i++) {
      console.log(`Número: ${i}`);
    }

    // Rango que excluye el límite superior
    for (let i = 0; i < 3; i++) {
      console.log(`Índice: ${i}`);
    }

    // Iterar sobre un array
    const frutas = ["Manzana", "Banana", "Cereza"];
    for (const fruta of frutas) {
      console.log(`Fruta: ${fruta}`);
    }

    // Iterar con índice usando .forEach() o .entries()
    frutas.forEach((fruta, indice) => {
      console.log(`${indice}: ${fruta}`);
    });

    // Iterar sobre un objeto
    const edades: Record<string, number> = { Ana: 25, Luis: 30, María: 28 };
    for (const [nombre, edad] of Object.entries(edades)) {
      console.log(`${nombre} tiene ${edad} años`);
    }
    ```
  </TabItem>
</Tabs>

### Usar stride para pasos personalizados

```swift
// Contar de 2 en 2
for numero in stride(from: 0, to: 10, by: 2) {
    print(numero)  // 0, 2, 4, 6, 8
}

// Cuenta regresiva
for numero in stride(from: 10, through: 0, by: -2) {
    print(numero)  // 10, 8, 6, 4, 2, 0
}
```

### Ignorar el valor del iterador

```swift
// Usar _ cuando no necesitas el valor
var potencia = 1
for _ in 1...10 {
    potencia *= 2
}
print("2^10 = \(potencia)")  // 1024
```

## Bucle repeat-while

Similar a `do-while` en otros lenguajes. Ejecuta el bloque al menos una vez.

```swift
var intentos = 0
repeat {
    intentos += 1
    print("Intento número \(intentos)")
} while intentos < 3

print("Total de intentos: \(intentos)")
```

## Guard

`guard` es una salida anticipada. Si la condición no se cumple, debe salir del ámbito actual.

<Tabs syncKey="lang">
  <TabItem label="Swift">
    ```swift
    func verificarEdad(_ edad: Int) {
        guard edad >= 18 else {
            print("Debes ser mayor de edad")
            return
        }
        guard edad <= 120 else {
            print("Edad no válida")
            return
        }
        print("Edad válida: \(edad) años. Bienvenido.")
    }

    verificarEdad(15)   // "Debes ser mayor de edad"
    verificarEdad(25)   // "Edad válida: 25 años. Bienvenido."
    verificarEdad(150)  // "Edad no válida"
    ```
  </TabItem>
  <TabItem label="React (Next.js)">
    ```typescript
    function verificarEdad(edad: number): void {
      if (edad < 18) {
        console.log("Debes ser mayor de edad");
        return;
      }
      if (edad > 120) {
        console.log("Edad no válida");
        return;
      }
      console.log(`Edad válida: ${edad} años. Bienvenido.`);
    }

    verificarEdad(15);   // "Debes ser mayor de edad"
    verificarEdad(25);   // "Edad válida: 25 años. Bienvenido."
    verificarEdad(150);  // "Edad no válida"
    ```
  </TabItem>
</Tabs>

<PlaygroundLink />

:::tip
Usa `guard` para validar condiciones al inicio de una función. Esto reduce la anidación y hace el código más legible, mostrando claramente el "camino feliz".
:::

## Control de flujo en bucles

```swift
// break: sale del bucle completamente
for i in 1...10 {
    if i == 5 {
        break
    }
    print(i)  // 1, 2, 3, 4
}

// continue: salta a la siguiente iteración
for i in 1...10 {
    if i % 2 == 0 {
        continue  // Salta los números pares
    }
    print(i)  // 1, 3, 5, 7, 9
}
```

## Resumen

| Estructura | Uso |
|-----------|-----|
| `if/else` | Decisiones condicionales |
| `switch` | Múltiples caminos con pattern matching |
| `for-in` | Iterar sobre secuencias |
| `while` | Repetir mientras se cumpla condición |
| `repeat-while` | Ejecutar al menos una vez |
| `guard` | Salida anticipada si no se cumple condición |
