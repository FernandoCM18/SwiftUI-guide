---
title: Publishers
description: Tipos de Publishers en Combine - Just, Future, PassthroughSubject y CurrentValueSubject.
---

Un **Publisher** es un tipo que emite una secuencia de valores a lo largo del tiempo. Puede emitir cero o más valores y terminar con una finalización exitosa o un error.

## Just

Emite un único valor y luego se completa:

```swift
import Combine

let publisher = Just(42)

publisher.sink { completion in
    print("Completado: \(completion)")
} receiveValue: { valor in
    print("Valor: \(valor)")
}
// Valor: 42
// Completado: finished
```

> [Probar en Swift Playground →](https://swiftfiddle.com/)

## Future

Emite un único valor de forma asíncrona en el futuro:

```swift
func obtenerUsuario(id: Int) -> Future<String, Error> {
    Future { promise in
        // Simular llamada de red
        DispatchQueue.main.asyncAfter(deadline: .now() + 1) {
            promise(.success("Usuario \(id)"))
        }
    }
}

let cancelable = obtenerUsuario(id: 1)
    .sink { completion in
        switch completion {
        case .finished:
            print("Completado")
        case .failure(let error):
            print("Error: \(error)")
        }
    } receiveValue: { usuario in
        print(usuario)
    }
```

> [Probar en Swift Playground →](https://swiftfiddle.com/)

## PassthroughSubject

Un publisher que puedes controlar manualmente, enviando valores a demanda. No almacena el último valor:

```swift
let subject = PassthroughSubject<String, Never>()

let cancelable = subject.sink { valor in
    print("Recibido: \(valor)")
}

subject.send("Hola")
subject.send("Mundo")
subject.send(completion: .finished)
// Recibido: Hola
// Recibido: Mundo
```

> [Probar en Swift Playground →](https://swiftfiddle.com/)

## CurrentValueSubject

Similar a `PassthroughSubject` pero almacena el valor actual:

```swift
let subject = CurrentValueSubject<Int, Never>(0)

print(subject.value) // 0

let cancelable = subject.sink { valor in
    print("Valor actual: \(valor)")
}

subject.send(1)
subject.send(2)

print(subject.value) // 2
```

> [Probar en Swift Playground →](https://swiftfiddle.com/)

## @Published

Una propiedad wrapper que crea automáticamente un publisher:

```swift
class Contador: ObservableObject {
    @Published var valor = 0
}

let contador = Contador()

let cancelable = contador.$valor
    .sink { valor in
        print("Contador: \(valor)")
    }

contador.valor = 1  // Contador: 1
contador.valor = 5  // Contador: 5
```

> [Probar en Swift Playground →](https://swiftfiddle.com/)

## Comparación

| Publisher | Valores | Almacena valor | Uso típico |
|---|---|---|---|
| `Just` | Uno | No | Valor inmediato |
| `Future` | Uno (async) | No | Operaciones asíncronas |
| `PassthroughSubject` | Múltiples | No | Eventos de UI |
| `CurrentValueSubject` | Múltiples | Sí | Estado mutable |
| `@Published` | Múltiples | Sí | Propiedades observables |

:::tip
Usa `PassthroughSubject` para eventos (como taps de botón) y `CurrentValueSubject` cuando necesites acceder al valor actual en cualquier momento.
:::
